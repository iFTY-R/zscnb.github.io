# 娱乐一下

# 基础要点

  1. 插值表达式：只能是单个表达式
   - {{'a'}} -- 字符串
   - {{1}}  -- 数字
   - {{true}} -- 布尔值
   - {{[1,2,3]}} -- 数组
   - {{ {a:1,b:10} }} -- 对象
   - 使用 JavaScript 表达式 
      - 所属类型都是运算表达式都可以: {{1-1+1*2 +'b'}}
      - {{ number + 1 }}
      - 三目运算符：{{ ok ? 'YES' : 'NO' }}
      - 链式调用：{{ message.split('').reverse().join('') }}
      - 单个表达式：<div v-bind:id="'list-' + id"></div>
   - {{undefined}} {{null}} 在页面上显示为空

注意：插入对象时，前后必须有空格，否则不显示

  1. 解析变量
  • 我们使用的数据，需要先在data中存在，否则没有定义
  • 数据要在data先存在，才能实现数据绑定，视图更新,否则不显示，注意如果绑定是和原数据相同的值，则不更新，它会把数据放在一个沙盒，如果不相同就更新，相同则不更新
  • 数组
    • 通过索引的方式和数组长度的方式改变数组，不能实现视图实时更新
    • 可以实现数组实时更新：数组变异方式： push、pop、shift、unshift、sort、reverse、splice
  • 对象
    • vm.$set(要修改对象，索引值，要插入的值)
    • $set不仅可以修改对象，还可以修改数组，字符串
  1. 实例方法
  • vm.$el 访问根元素
  • 更改元素是异步的，js会首先同步任务，后执行异步任务，要想数据和视图一样，用
vm.$nextTick(() => {
  //要修改的数据同步
})

new Vue({
  // ...
  methods: {
    // ...
    example: function () {
      // 修改数据
      this.message = 'changed'
      // DOM 还没有更新
      this.$nextTick(function () {
        // DOM 现在更新了
        // `this` 绑定到当前实例
        this.doSomethingElse()
      })
    }
  }
})
  • vm.$mount() 返回实例本身

# vue指令

  1. v-html
  • 更新元素的 innerHTML
  • 注意：内容按普通 HTML 插入 - 不会作为 Vue 模板进行编译
  • 在网站上动态渲染任意 HTML 是非常危险的，因为容易导致 XSS 攻击。只在可信内容上使用 v-html，永不用在用户提交的内容上。 在单文件组件里，scoped 的样式不会应用在 v-html 内部，因为那部分 HTML 没有被 Vue 的模板编译器处理。如果你希望针对 v-html 的内容设置带作用域的 CSS，你可以替换为 CSS Modules 或用一个额外的全局style元素手动设置类似 BEM 的作用域策略。
  • 示例：
<div v-html="html"></div>
  1. v-text
  • 更新元素的 textContent。如果要更新部分的 textContent ，需要使用 插值。
  • 示例：
<span v-text="msg"></span>//会把文本覆盖掉，显示v-text的内容
<!-- 和下面的一样 -->
<span>{{msg}}</span> 插值表达式不会覆盖文本
  1. v-if
  • 在切换时元素及它的数据绑定 / 组件被销毁并重建。如果元素是 template ，将提出它的内容作为条件块。
  • 多使用template,页面上没有template. 提高性能
  • 示例：
  <div id="app">
        <template v-if="flag">
            <div>赵寿聪助手</div>
        </template>
    </div>
    <script>
        const vm = new Vue({
            el: '#app',
            data: {
                flag: !true
            }
        })
    </script>
  1. v-else
  • 前一兄弟元素必须有 v-if 或 v-else-if。不能第三者插入你的家庭，否则不正确 (v-if,v-else-if, v-else)
  • 示例：
<div v-if="Math.random() > 0.5">
  Now you see me
</div>
<div v-else>
  Now you don't
</div>
  1. v-else-if
  • 前一兄弟元素必须有 v-if 或 v-else-if
<div v-if="type === 'A'">
  A
</div>
<div v-else-if="type === 'B'">
  B
</div>
<div v-else-if="type === 'C'">
  C
</div>
<div v-else>
  Not A/B/C
</div>
  1. v-show
  • 根据表达式之真假值，切换元素的 display CSS 属性。
  • v-show和v-if区别
    • v-show 元素真实存在，只是切换display的none和block属性值
    • v-if 完全移除添加元素，在页面上用注释替代，表示有一个元素
    • v-show和template不生效 和 v-if和template生效
    • 如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。
  1. v-bind: 缩写 ：绑定属性

用法：

  • 动态地绑定一个或多个特性，或一个组件 prop 到表达式。

  • 在绑定 class 或 style 特性时，支持其它类型的值，如数组或对象。可以通过下面的教程链接查看详情。

  • 在绑定 prop 时，prop 必须在子组件中声明。可以用修饰符指定不同的绑定类型。

  • 没有参数时，可以绑定到一个包含键值对的对象。注意此时 class 和 style 绑定不支持数组和对象。

示例：

<!-- 绑定一个属性 -->
<img v-bind:src="imageSrc">

<!-- 动态特性名 (2.6.0+) -->
<button v-bind:[key]="value"></button>
 
<!-- 缩写 -->
<img :src="imageSrc">

<!-- 动态特性名缩写 (2.6.0+) -->
<button :[key]="value"></button>

<!-- 内联字符串拼接 -->
<img :src="'/path/to/images/' + fileName">

<!-- class 绑定 --> :class权重大于class的权重
<div :class="{ red: isRed }"></div>
<div :class="[classA, classB]"></div>
<div :class="[classA, { classB: isB, classC: isC }]">

<!-- style 绑定 --> :style权重大于style的权重
1. 可以使用驼峰式(camelCase): 如 fontSize
2. 短横线分隔(kebab-case),记得用单引号括起来： 如 'font-size'
<div :style="{ fontSize: size + 'px' }"></div>
<div :style="[styleObjectA, styleObjectB]"></div>

<!-- 绑定一个有属性的对象 -->
<div v-bind="{ id: someProp, 'other-attr': otherProp }"></div>

<!-- 通过 prop 修饰符绑定 DOM 属性 -->
<div v-bind:text-content.prop="text"></div>

<!-- prop 绑定。“prop”必须在 my-component 中声明。-->
<my-component :prop="someThing"></my-component>

<!-- 通过 $props 将父组件的 props 一起传给子组件 -->
<child-component v-bind="$props"></child-component>

<!-- XLink -->
<svg><a :xlink:special="foo"></a></svg>
  1. v-on 简写 @

注意参数问题

  • 如果该方法不需要额外参数，那么方法后的小括号()可以不添加

  • 注意：如果方法本身中有一个参数且调用方法没有传参数，那么会默认将原生事件event参数传递进去

  • 在监听原生 DOM 事件时，方法以事件为唯一的参数。如果使用内联语句，语句可以访问一个 $event 属性：v-on:click="handle('ok', $event)"。

修饰符：

  • .stop - 调用 event.stopPropagation()。
  • .prevent - 调用 event.preventDefault()。
  • .capture - 添加事件侦听器时使用 capture 模式。
  • .self - 只当事件是从侦听器绑定的元素本身触发时才触发回调。
  • .{keyCode | keyAlias} - 只当事件是从特定键触发时才触发回调。
  • .native - 监听组件根元素的原生事件。
  • .once - 只触发一次回调。
  • .left - (2.2.0) 只当点击鼠标左键时触发。
  • .right - (2.2.0) 只当点击鼠标右键时触发。
  • .middle - (2.2.0) 只当点击鼠标中键时触发。
  • .passive - (2.3.0) 以 { passive: true } 模式添加侦听器

注意：避免变量重名，vue会先去data上找，再去别处找

  1. v-for

预期： Array | Object | number | string | Iterable (2.6 新增)

:key是唯一的

  • 使用index值可能造成key值不唯一
  • 我们从后台获取的数据通常都有id值:key = id
<div v-for="item in items" :key="item.id">
  {{ item.text }}
</div>

<!-- 数组 -->
<div v-for="(item, index) in items"></div>
<!-- 对象 -->
<div v-for="(val, key) in object"></div>
<div v-for="(val, name, index) in object"></div>

v-for 的默认行为会尝试原地修改元素而不是移动它们。

要强制其重新排序元素，你需要用特殊属性key 来提供一个排序提示：

  • 插入元素，插入的位置后面依次会向后移几位，性能不好
<div v-for="item in items" :key="item.id">
  {{ item.text }}
</div>
  • v-for可以跟template连用，当遍历对象不需要在页面添加新的元素，用template替代,我们可以看到template标签不在了
     <template v-for="key in 8">
           {{key}}
      </template>

  1. v-once

注意：只渲染元素和组件一次，此后不随数据更改视图发生改变

<!-- 单个元素 -->
<span v-once>This will never change: {{msg}}</span>
<!-- 有子元素 -->
<div v-once>

<!-- 组件 -->

<!-- `v-for` 指令-->

  1. v-pre

用法：

跳过这个元素和它的子元素的编译过程。可以用来显示原始 Mustache(胡须或双大括号) 标签。跳过大量没有指令的节点会加快编译。

示例：

<span v-pre>{{ this will not be compiled }}</span>

浏览器看到效果：

  1. v-cloak

cloak: 英译：斗篷:意思隐藏的意思

原因

  1. 浏览器突然出现卡顿，会出现带双大括号的原始标签
  2. 同步代码执行后，遇到异步代码，浏览器会先执行同步后异步
  3. 把js代码写在head里面，就不会出现这个问题

用法 可以隐藏未编译的 Mustache 标签直到实例准备完毕。

示例：

[v-cloak] {
  display: none;
}

<div v-cloak>
  {{ message }}
</div>

不会显示，直到编译结束。

# 计算属性

  • 计算属性是有缓存的
  • 计算属性默认只有getter，不过在需要时你也可以提供一个 setter
  • 直接写变量名，不用写函数，如下所示
<div id="demo">{{ fullName }}</div>

var vm = new Vue({
  el: '#demo',
  data: {
    firstName: 'Foo',
    lastName: 'Bar'
  },
  computed: {
    //计算属性getter方法  扼要写法
    fullName: function () {
      return this.firstName + ' ' + this.lastName
    }
  }
})

// ...
computed: {
  //计算属性的完整写法
  fullName: {
    // getter
    get: function () {
      return this.firstName + ' ' + this.lastName
    } ,
    // setter
    set: function (newValue) {
      var names = newValue.split(' ')
      this.firstName = names[0]
      this.lastName = names[names.length - 1]
    }
  }
}
// ...

计算属性缓存 vs 方法

  • 计算属性是基于它的依赖进行更新的，只有在相关依赖发送改变时才能更新变化
  • 计算属性是有缓存的，只要相关依赖没有改变，多次访问计算属性得到的值是之前的缓存的结果，不会多次执行
  • 每当触发重新渲染时，调用方法将总会再次执行函数。
  • 如果你不希望有缓存，请用方法来替代。

# 过滤器

  • Vue.js 允许你自定义过滤器，可被用于一些常见的文本格式化
  • 过滤器可以用在两个地方：双花括号插值和 v-bind 表达式 (后者从 2.1.0+ 开始支持)。
  • 过滤器应该被添加在 JavaScript 表达式的尾部，由“管道 ←→ |”符号指示：
  • 过滤器函数总接收表达式的值 (之前的操作链的结果) 作为第一个参数
  • 当全局过滤器和局部过滤器重名时，会采用局部过滤器。
<!-- 在双花括号中 --> 
{{ message | capitalize }}

<!-- 在 `v-bind` 中 -->
<div v-bind:id="rawId | formatId"></div>

局部过滤器

filters: {
  // capitalize 过滤器函数将会收到 message 的值作为第一个参数。
  capitalize: function (value) {
    if (!value) return ''
    value = value.toString()
    return value.charAt(0).toUpperCase() + value.slice(1)
  }
}

全局过滤器

Vue.filter('capitalize', function (value) {
  if (!value) return ''
  value = value.toString()
  return value.charAt(0).toUpperCase() + value.slice(1)
})

new Vue({
  // ...
})

过滤器可以串联：

{{ message | filterA | filterB }}

在这个例子中，filterA 被定义为接收单个参数的过滤器函数，表达式 message 的值将作为参数传入到函数中。然后继续调用同样被定义为接收单个参数的过滤器函数 filterB，将filterA 的结果传递到 filterB 中。

过滤器是 JavaScript 函数，因此可以接收参数：

{{ message | filterA('arg1', arg2) }}

这里，filterA 被定义为接收三个参数的过滤器函数。其中 message 的值作为第一个参数，普通字符串 'arg1' 作为第二个参数，表达式arg2的值作为第三个参数。

# 组件化

组件化是指把整体页面拆分成小小的、可复用组件，便于对组件进行管理维护和扩展功能

注册组件的基本步骤：

  1. 创建组件构造器

调用Vue.extend( options )方法创建组件构造器

  1. 注册组件

调用Vue.component( id, [definition] ))方法注册组件

  1. 使用组件

在Vue实例的作用范围内使用组件

示例如下图：

点击查看代码

# 组件data必须是一个函数

Object是引用数据类型，如果不用function返回，每个组件的data都是内存的同一个地址，一个数据改变了其他也改变了；

  • JavaScript只有函数构成作用域
  • 注意理解作用域，只有函数{}构成作用域,对象的{}以及if(){}都不构成作用域
  • data是一个函数时，每个组件实例都有自己的作用域，每个实例相互独立，不会相互影响。

示例

点击查看代码 
     const obj = {
            name: 'zsc',
            age: 18
        }

        function test() {
            return obj
        }

        const A1 = test()
        const A2 = test()
        const A3 = test()

        // 三个函数调用都指向同一个内存地址
        A1.name = '赵寿聪'
        console.log(A2.name) //'赵寿聪'
        console.log(A3.name) // '赵寿聪'

如果三个实例同时引用一个对象，那么当你修改其中一个属性的时候，另外一个实例也会跟着改变；

两个实例应该有自己各自的域才对，需要通过下面的方法进行处理

    function test() {
            return {
                name: 'zsc',
                age: 18
            }
        }

        const A1 = test()
        const A2 = test()
        const A3 = test()

        A1.name = '赵寿聪'
        // 在函数作用域每次调用都返回不同对象，它们内存地址不一样
        console.log(A2.name)//'zsc'
        console.log(A3.name)// 'zsc'

正如官网说的那样，每个实例可以维护一份被返回对象的独立的拷贝(相互独立，互不干扰的可复用组件)

props基本用法:

  1. 字符串数组传值

  1. 对象props验证传值
Vue.component('my-component', {
  props: {
    // 基础的类型检查 (`null` 和 `undefined` 会通过任何类型验证)
    propA: Number,
    // 多个可能的类型
    propB: [String, Number],
    // 必填的字符串
    propC: {
      type: String,
      required: true
    },
    // 带有默认值的数字
    propD: {
      type: Number,
      default: 100
    },
    // 带有默认值的对象
    propE: {
      type: Object,
      // 对象或数组默认值必须从一个工厂函数获取
      default: function () {
        return { message: 'hello' }
      }
    },
    // 自定义验证函数
    propF: {
      validator: function (value) {
        // 这个值必须匹配下列字符串中的一个
        return ['success', 'warning', 'danger'].indexOf(value) !== -1
      }
    }
  }
})

# 单向数据流

所有的 prop 都使得其父子 prop 之间形成了一个单向下行绑定：父级 prop 的更新会向下流动到子组件中，但是反过来则不行。这样会防止从子组件意外改变父级组件的状态，从而导致你的应用的数据流向难以理解。

额外的，每次父级组件发生更新时，子组件中所有的 prop 都将会刷新为最新的值。这意味着你不应该在一个子组件内部改变 prop。如果你这样做了，Vue 会在浏览器的控制台中发出警告。

这里有两种常见的试图改变一个 prop 的情形：

这个 prop 用来传递一个初始值；这个子组件接下来希望将其作为一个本地的 prop 数据来使用。在这种情况下，最好定义一个本地的 data 属性并将这个 prop 用作其初始值：

props: ['initialCounter'],
data: function () {
  return {
    counter: this.initialCounter
  }
}

这个 prop 以一种原始的值传入且需要进行转换。在这种情况下，最好使用这个 prop 的值来定义一个计算属性：

props: ['size'],
computed: {
  normalizedSize: function () {
    return this.size.trim().toLowerCase()
  }
}

注意

在 JavaScript 中对象和数组是通过引用传入的，所以对于一个数组或对象类型的 prop 来说，在子组件中改变这个对象或数组本身将会影响到父组件的状态。

# 父子组件通信案例

范例展示

    <div id="app">
        <my-cpn :number1="num1" :number2="num2" @num1change="num1change" @num2change="num2change" />
    </div>

    <!-- 子组件 -->
    <template id="demo">
        <div>
            <h2>num1父传子props:{{number1}}</h2>
            <h2>dnum1子组件data:{{dnum1}}</h2>
            <input type="number" :value="dnum1" @input="num1Change">
            <h2>num2父传子props:{{number2}}</h2>
            <h2>dnum2子组件data:{{dnum2}}</h2>
            <input type="number" :value="dnum2" @input="num2Change">
        </div>
    </template>
    <script>
        // 父组件
        new Vue({
            el: '#app',
            data: {
                num1: 1,
                num2: 2
            },
            methods: {
                num1change(value) {
                    // value是string  加"+" 转换数字类型
                    this.num1 = +value
                },
                num2change(value) {
                    this.num2 = +value
                },
            },
            components: {
                'myCpn': {
                    template: '#demo',
                    data() {
                        // 所有的 prop 都使得其父子 prop 之间形成了一个单向下行绑定：
                        // 父级 prop 的更新会向下流动到子组件中，但是反过来则不行。
                        // 这样会防止从子组件意外改变父级组件的状态，从而导致你的应用的数据流向难以理解。
                        return {
                            dnum1: this.number1,
                            dnum2: this.number2
                        }
                    },
                    props: {
                        number1: Number,
                        number2: Number
                    },
                    methods: {
                        num1Change(event) {
                            // 1.将input中的value赋值到dnum1中
                            this.dnum1 = event.target.value

                            // 2. 为了让父组件可以修改num1值，发出一个事件
                            this.$emit('num1change', this.dnum1)

                            // 3. 同时修改dnum2的值
                            this.dnum2 = this.dnum1 * 100

                            // 4. 为了让父组件可以修改num2值，发出一个事件
                            this.$emit('num2change', this.dnum2)
                        },
                        num2Change(event) {
                            this.dnum2 = event.target.value
                            this.$emit('num2change', this.dnum2)
                            this.dnum1 = this.dnum2 / 100
                            this.$emit('num1change', this.dnum1)
                        }
                    },
                }
            }

        })
    </script>

  1. 登录切换的input复用问题

当用户输入账户名时，而又想不起来账户名，切换类型，我们可以看到用户输入的还在

如何解决这个问题呢？

问题解答

  • 这是因为Vue在进行虚拟DOM渲染时，处于性能考虑，会尽可能的复用已经存在的元素，而不是重新创建新的元素
  • 在上面的案例中， Vue内部会发现原来的input元素不再使用，直接做为else中的input来使用已经存在的内容

解决方案

  • 如果我们不希望Vue出现类似重复利用的问题，可以给对应的input添加key
  • 并且我们需要保证key的不同

            }

</body>
