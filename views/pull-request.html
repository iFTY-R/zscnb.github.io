新手不得不会提交代码(PR)

Mr.Zhao 2020-03-26 14:27:56 必会 技能

github、gitee越来越为大家所熟悉。如果发现了某个项目的BUG，也发扬一下开源精神，将修正后的代码提交给原作者。那么，本文讲得就是如何将修改后的代码向github/gitee提交。

总的流程如下：

  1. 先 Fork 对应的项目，相当于拷贝一份到自己的github账户下

  2. 然后再clone 到本地分支，对bug做相应的修改

3.发起pull request 给原作者，让他看到，并决定是否接受你的修改

  1. 原作者同意接受这个bug，就会 merge 到他自己的项目中

  2. 提交流程结束了

  • 方法/步骤

Github 官方文档的定义如下:

"Pull Request 是一种通知机制。你修改了他人的代码，将你的修改通知原来的作者，希望他合并你的修改，这就是 Pull Request。"

Pull Request 本质上是一种软件的合作方式，是将涉及不同功能的代码，纳入主干的一种流程。这个过程中，还可以进行讨论、审核和修改代码。

以 Github 上的的 bootstrap为例，实操如下：

# 方法/步骤

# 1. 先 fork 相应的项目，相当于拷贝一份到自己的github账户下

# 2. 在自己本地 git clone 这个项目，修改bug

git clone https://github.com/beepony/bootstrap.git

cd bootstrap

git checkout -b test-pr

git add . && git commit -m "test-pr"

git push origin test-pr

不能有帮助，请移驾到官方文档

# 3.完成提交后，来到自己对应仓库项目中

  1. 点击New pull request绿色按钮

  2. 选择自己的要推送的分支 例如如下图所示：test-pr

注意：源分支一定自己的账户名，目标分支一定是原作者账户名！如果不是，请重做上面的步骤

# 4. 添加对bug的注释信息，确认提交

# 5. 如果原作者同意接受这个bug，就会 merge 到他自己的项目中 OK!
